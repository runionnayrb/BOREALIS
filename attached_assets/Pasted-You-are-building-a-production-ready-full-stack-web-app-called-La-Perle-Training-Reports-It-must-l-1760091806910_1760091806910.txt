You are building a production ready full stack web app called “La Perle Training Reports.” It must look native on desktop and mobile, feel fast, and export beautiful paginated PDFs.

Tech choices

• Frontend: React with Vite, React Router, TypeScript, Tailwind CSS for a clean responsive UI
• State: React Query for server cache and optimistic updates, Zod for client validation
• Rich text: TipTap editor with bullet and number lists, alignment controls, font size in px
• Date time: date fns with timezone via Intl API using Asia Dubai
• Backend: Node, Express, TypeScript
• Database: SQLite with Prisma ORM
• PDF: server side export using Playwright for high fidelity printing and CSS page breaks. Also provide client print styles as a fallback
• Auth: simple passcode gate now, prepared for JWT later
• Testing: a small Playwright test that confirms header date format and pagination

Core data model with Prisma

Implement these entities and relationships. Use singular table names. All entities include createdAt and updatedAt.

Act
• id, name, orderIndex
• many Departments through ActDepartment
• many Artists through ActArtist
• visibility and ordering controls in settings

Department
• id, name, orderIndex
• many Technicians
• belongs to many Acts through ActDepartment

ArtistGroup
• id, name, orderIndex
• many Artists

Artist
• id, firstName, lastName, stageName optional, orderIndex
• belongs to zero or one ArtistGroup
• belongs to many Acts through ActArtist

Technician
• id, firstName, lastName, role optional, orderIndex
• belongs to one Department
• can be assigned to positions on a Training

Report
• id, title, createdAt, updatedAt
• headerLeftImageUrl optional
• headerMiddleTitle default “Training Report”
• headerRightImageUrl optional
• headerDateString stored as a formatted string
• richBody JSON from TipTap

Training
• id, reportId FK
• actId FK
• startTime ISO string
• endTime ISO string
• durationMinutes computed on save and exposed by API
• departmentPositions array of entries like
{ departmentId, leadTechnicianId, notes optional }
Only departments assigned to the chosen act may appear
For the lead dropdown only technicians from that department are selectable
• scheduledFor DateTime optional to support future schedule integration. This indicates when the training is planned to occur in local time for Asia Dubai
• scheduleId String optional to reference an external schedule item later

ActDepartment join table
• id, actId, departmentId

ActArtist join table
• id, actId, artistId

API design with Zod validation under  /api

General
• All list endpoints support pagination and basic search where applicable
• Validate inputs with identical Zod schemas on client and server

Settings
• Acts: list, create, update, delete, reorder
• Departments: list, create, update, delete, reorder
• ArtistGroups: list, create, update, delete, reorder
• Artists: list, create, update, delete, reorder
• Technicians: list, create, update, delete, reorder
• Assignment endpoints
POST /api/acts/:id/assignDepartments accepts array of departmentIds
POST /api/acts/:id/assignArtists accepts array of artistIds
POST /api/artistGroups/:id/assignArtists accepts array of artistIds

Reports
• GET /api/reports list with pagination and search
• POST /api/reports creates a new report with default headerMiddleTitle and headerDateString formatted as “EEEE, MMMM d, yyyy” using Asia Dubai timezone. Example “Thursday, October 9, 2025”
• GET /api/reports/:id
• PUT /api/reports/:id updates header images, title, rich body
• DELETE /api/reports/:id

Trainings
• POST /api/trainings creates a training for a report and act with startTime and endTime. Compute durationMinutes as end minus start
• PUT /api/trainings/:id updates start or end and recomputes durationMinutes
• PUT /api/trainings/:id/positions updates departmentPositions. Server must validate that departmentId is assigned to the selected act and that leadTechnicianId belongs to that department
• GET /api/trainings/:id
• DELETE /api/trainings/:id

Scheduling friendly filters
• GET /api/trainings?date=YYYY MM DD returns trainings whose scheduledFor falls on that calendar day in Asia Dubai
• GET /api/trainings?from=YYYY MM DD&to=YYYY MM DD returns trainings whose scheduledFor falls within the inclusive date range in Asia Dubai
• All training create and update endpoints accept optional scheduledFor and scheduleId. If scheduledFor is omitted it remains null

PDF
• GET /api/reports/:id/pdf returns a PDF rendered server side with the exact print layout
• Use consistent margins and CSS page break rules
• Include page numbers in the footer
